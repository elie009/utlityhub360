## 🤖 How Auto-Recurring Bills Work

### The Problem We Solved
❌ **Before:** You had to manually enter every bill every month  
✅ **Now:** Enable auto-generation once, and the system creates ALL 12 MONTHS!

### Two Modes of Auto-Generation

#### **Mode 1: 12-Month Auto-Generation** ⭐⭐ NEW!
Perfect for planning ahead or fixed monthly bills

**Setup Once:**
```
User creates Meralco bill in October
├── Amount: $3,050
├── Due Date: Oct 10
├── Frequency: Monthly
└── Auto-Generate: ✅ ENABLED ⭐

System IMMEDIATELY creates ALL 12 months:
├── ✅ October 2025: $3,050 (original)
├── ✅ November 2025: $3,050 (auto-generated)
├── ✅ December 2025: $3,050 (auto-generated)
├── ✅ January 2026: $3,050 (auto-generated)
├── ... (all months through December)
└── ✅ September 2026: $3,050 (auto-generated)

All bills created with:
├── Same base amount ($3,050)
├── Same due day (10th of each month)
├── Status: PENDING
├── IsAutoGenerated: true
└── Notes: "Auto-generated for [Month Year]"
```

**Update Individual Months:**
```
GET /api/bills/monthly?year=2025&month=11
# Returns all bills for November 2025

PUT /api/bills/{novemberBillId}/monthly
{
  "amount": 3200.00,
  "notes": "Updated to actual amount"
}
# Updates only November's bill
```

**Benefits:**
- ✅ **Instant Planning:** See your entire year of bills at once
- ✅ **Flexible Updates:** Modify individual months as needed
- ✅ **No Missed Bills:** All 12 months created immediately
- ✅ **Great for:** Fixed bills, annual planning, budgeting

## 🧭 1. Purpose

The Variable Monthly Billing feature allows users to:

- **Track historical bills** - View and analyze past bills over time
- **Calculate averages** - Understand typical monthly costs using simple or weighted averages
- **Forecast future bills** - Predict next month's expenses based on historical data
- **Monitor variance** - Identify when bills are higher or lower than expected
- **Set budgets** - Create spending limits and receive alerts when exceeded
- **Visualize trends** - See usage patterns through graphs and charts

### Step 3 – System Computes Average and Forecast

**Purpose:** To estimate next month's bill and detect budget trends based on historical data.

**🔌 API Implementation:**
```http
# Get complete analytics with all calculation methods
GET /api/bills/analytics/history?provider=Meralco&billType=utility&months=6
Authorization: Bearer {token}
```

#### Option 1: Simple Average (Last 3 Months)

**Formula:**
```
Estimated Bill = (Month1 + Month2 + Month3) / 3
```

**Example Calculation:**
```
Estimated Bill = (July + August + September) / 3
               = ($3,200 + $2,870 + $2,640) / 3
               = $8,710 / 3
               = $2,903.33
```

**🔌 API Call:**
```http
GET /api/bills/analytics/forecast?provider=Meralco&billType=utility&method=simple
Authorization: Bearer {token}
```

**Use Case:** Best for stable usage patterns with minimal seasonal variation.

#### Option 2: Weighted Average (Recent Months Count More)

**Formula:**
```
Estimated Bill = (Month1 × Weight1) + (Month2 × Weight2) + (Month3 × Weight3)
Where: Weight1 + Weight2 + Weight3 = 1.0
```

**Example Calculation:**
```
Weights: Most Recent = 0.5, Middle = 0.3, Oldest = 0.2

Estimated Bill = ($3,200 × 0.5) + ($2,870 × 0.3) + ($2,640 × 0.2)
               = $1,600 + $861 + $528
               = $2,989
```

**🔌 API Call:**
```http
GET /api/bills/analytics/forecast?provider=Meralco&billType=utility&method=weighted
Authorization: Bearer {token}
```

**✅ Response:**
```json
{
  "success": true,
  "data": {
    "estimatedAmount": 2989.00,
    "calculationMethod": "weighted",
    "confidence": "medium",
    "estimatedForMonth": "2025-11-01T00:00:00Z",
    "recommendation": "Based on historical patterns, expect around $2,989.00 for next month."
  }
}
```

**Use Case:** Best for fluctuating usage patterns where recent trends are more predictive.

#### Option 3: Seasonal Average (12-Month Rolling)

**Formula:**
```
Estimated Bill = Average of same month from previous years
```

**Example Calculation:**
```
October Bills:
- Oct 2023: $2,950
- Oct 2024: $3,100

Estimated October 2025 = ($2,950 + $3,100) / 2 = $3,025
```

**Use Case:** Best for bills with strong seasonal patterns (e.g., heating in winter).

**System Action:**
- Automatically calculates all three averages
- Stores the estimated amount for the next billing period
- Updates the dashboard with forecast information
- Allows user to select preferred calculation method in settings

### Step 4 – Compare Actual vs. Estimated

When the October bill arrives at **$3,050**, the system compares it against the estimated amount.

**Variance Calculation:**
```
Variance = Actual Bill - Estimated Bill
Variance = $3,050 - $2,989 = +$61

Percentage Variance = (Variance / Estimated Bill) × 100
                    = ($61 / $2,989) × 100
                    = +2.04%
```

**🔌 API Implementation:**
```http
GET /api/bills/{billId}/variance
Authorization: Bearer {token}
```

**✅ Response:**
```json
{
  "success": true,
  "data": {
    "billId": "bill-123",
    "actualAmount": 3050.00,
    "estimatedAmount": 2989.00,
    "variance": 61.00,
    "variancePercentage": 2.04,
    "status": "slightly_over",
    "message": "Your bill is slightly higher than expected (+2.04%)",
    "recommendation": "Your bill is slightly above average. Monitor usage to keep costs down."
  }
}
```

**System Messages Based on Variance:**

| Variance | Message |
|----------|---------|
| **Over Budget** (+5% or more) | ⚠️ "Your electricity bill is $61 higher than expected (+2.04%). Consider reviewing your usage." |
| **Slightly Over** (+1% to +5%) | ℹ️ "Your bill is slightly higher than average this month." |
| **On Target** (-1% to +1%) | ✅ "Your bill is right on track with your average!" |
| **Under Budget** (-5% or less) | 💚 "Great job! Your bill is $50 lower than expected. You saved money this month!" |

### Step 6 – Alerts and Adjustments

The system monitors bills and sends alerts based on configured thresholds.

#### Alert Types

**1. Budget Exceeded Alert**
```
Trigger: Current bill exceeds user-defined budget
Message: "⚠️ Budget Alert: You exceeded your electricity budget by $50."
Action: Suggest budget adjustment or usage reduction tips
```

**2. Trend Increase Alert**
```
Trigger: Last 3 bills increase by more than 5% consecutively
Message: "📈 Trend Alert: Your electric bill increased by 6% compared to last month. 
         Consider adjusting your monthly budget to $3,100."
Action: Offer to auto-adjust budget
```

**3. Unusual Spike Alert**
```
Trigger: Bill is 20% or more above the 6-month average
Message: "🚨 Unusual Bill: Your electricity bill is 18% higher than your 6-month average. 
         This could indicate a meter issue or unusual usage."
Action: Suggest contacting the provider
```

**4. Due Date Reminder**
```
Trigger: 3 days before due date
Message: "⏰ Payment Reminder: Your Meralco bill of $3,050 is due in 3 days (Oct 10)."
Action: Link to mark as paid
```

**5. Overdue Alert**
```
Trigger: Bill past due date
Message: "❗ Overdue: Your Meralco bill of $3,050 was due on Oct 10. 
         Pay now to avoid late fees."
Action: Link to payment options
```

**6. Savings Achievement Alert**
```
Trigger: Bill is 10% or more below average
Message: "🎉 Great Job! You saved $290 on your electricity bill this month!"
Action: Share savings tips
```

| **User adds bill** | System records amount, date, provider, and status |
| **System updates history** | Keeps past 6–12 months for analysis and forecasting |
| **System computes average** | Uses simple, weighted, or seasonal average based on user preference |
| **System generates forecast** | Predicts next month's bill using selected calculation method |
| **System compares actual vs. estimated** | Calculates variance when new bill is entered |
| **System sends notifications** | Alerts user based on variance, budget, and due dates |
| **System visualizes data** | Displays trends, graphs, and insights on dashboard |
| **User marks as paid** | Updates status and tracks payment history |
